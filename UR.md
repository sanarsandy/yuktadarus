Berikut adalah Dokumen Perancangan Aplikasi (Software Design Document) untuk proyek One Day One Juz (ODOJ) Mobile.1. Ringkasan ProyekMembangun aplikasi pelacak tilawah Quran harian berbasis komunitas dengan target penyelesaian 30 Juz selama bulan Ramadan (atau harian umum). Fokus utama pada user experience yang minimalis dan fitur motivasi berbasis gamification.Tech StackFrontend: Nuxt.js 3 (Vue.js 3) + Tailwind CSS.Mobile Wrapper: Capacitor.js (untuk akses fitur native & distribusi app store).Backend: Golang (Gin atau Fiber framework).Database: PostgreSQL (untuk data relasional) & Redis (untuk caching/streak).Authentication: JWT (JSON Web Token).2. Analisis Fitur (Berdasarkan Gambar)Berdasarkan tangkapan layar yang Anda berikan, berikut fitur yang harus diimplementasikan:Daily Tracking: Tombol satu ketuk "Mark Juz Complete".Spiritual Motivation: Kutipan ayat harian (Daily Verse) yang berganti setiap hari.Progress Dashboard:Heatmap/Calendar: Visualisasi 30 hari tilawah.Statistik: Persentase penyelesaian, total Juz, dan Streak counter (api).Gamification (Badges): Sistem reward untuk pencapaian tertentu (Juz pertama, 1 minggu berturut-turut, dll).Social Referral: Fitur berbagi kode referal untuk mengajak orang lain (Share the Reward).3. Arsitektur Data (Database Schema)Table: usersColumnTypeDescriptionidUUID (PK)Unique ID usernameStringNama lengkapemailStringEmail unikreferral_codeString (6)Kode unik (contoh: 4R9L69)created_atTimestampWaktu daftarTable: tilawah_logsColumnTypeDescriptionidBigIntPKuser_idUUID (FK)Relasi ke userjuz_numberIntegerNo Juz (1-30)completed_atTimestampWaktu klik selesaiTable: badgesColumnTypeDescriptionidIntegerPKnameStringNama badge (e.g., "Khatam Complete")requirement_typeStringLogic trigger (streak/total_juz)4. Perancangan API (Golang)MethodEndpointDescriptionPOST/api/auth/registerPendaftaran user baruGET/api/dashboardMengambil status hari ini, streak, dan ayat harianPOST/api/tilawah/completeInput penyelesaian JuzGET/api/progressData untuk kalender heatmap (30 hari)GET/api/badgesDaftar badge yang sudah dan belum didapat5. Implementasi Frontend (Nuxt + Capacitor)UI/UX StrategyTheme: Gunakan variabel warna CSS untuk warna Hijau Emerald (#2D5A43) dan Background Cream Off-white (#F9F7F2) agar sesuai gambar.Components: Buat komponen kartu dengan box-shadow berlapis untuk efek Neumorphism.State Management: Gunakan Pinia untuk menyimpan data user dan progress secara lokal agar aplikasi terasa instan (optimistic UI).Integrasi CapacitorHaptics: Tambahkan efek getar saat user menekan "Mark Juz Complete".Share API: Gunakan @capacitor/share untuk fitur "Copy Invite Link" agar muncul dialog share native Android/iOS.Local Notifications: Pengingat harian jika user belum mencentang Juz pada jam tertentu.6. Roadmap PengembanganPhase 1: MVP (Minimum Viable Product)Setup project Nuxt & Golang.Fitur Login/Register.Tombol "Mark Juz Complete" dan progress sederhana (1/30).Phase 2: Gamification & UIImplementasi sistem Badge.Pembuatan kalender heatmap.Integrasi ayat harian (API Quran).Phase 3: Mobile & SocialBuild ke Android/iOS menggunakan Capacitor.Fitur Referral & Share.Push Notification pengingat tilawah.
